# Руководство по внесению вклада

Спасибо за интерес к проекту! Мы рады любому вкладу.

## Как внести вклад

### 1. Сообщить об ошибке

Если вы нашли ошибку:
- Проверьте, не была ли она уже сообщена в [Issues](https://github.com/Fixcat/UbuntuAutoUpdater/issues)
- Создайте новый Issue с подробным описанием
- Укажите версию Ubuntu и шаги для воспроизведения

### 2. Предложить улучшение

Есть идея? Создайте Issue с тегом "enhancement" и опишите:
- Что вы хотите улучшить
- Почему это будет полезно
- Как это должно работать

### 3. Внести код

#### Процесс:

1. **Fork репозитория**
   ```bash
   # Нажмите кнопку Fork на GitHub
   ```

2. **Клонируйте свой fork**
   ```bash
   git clone https://github.com/ваш-username/UbuntuAutoUpdater.git
   cd UbuntuAutoUpdater
   ```

3. **Создайте ветку**
   ```bash
   git checkout -b feature/ваша-функция
   ```

4. **Внесите изменения**
   - Следуйте стилю кода проекта
   - Добавьте комментарии на русском языке
   - Тестируйте на Ubuntu

5. **Закоммитьте**
   ```bash
   git add .
   git commit -m "Добавлена функция: описание"
   ```

6. **Отправьте в свой fork**
   ```bash
   git push origin feature/ваша-функция
   ```

7. **Создайте Pull Request**
   - Опишите изменения
   - Укажите связанные Issues

## Стандарты кода

### Bash скрипты:
- Используйте `#!/bin/bash` в начале
- Добавляйте комментарии на русском
- Проверяйте код с помощью `shellcheck`
- Используйте двойные кавычки для переменных: `"$variable"`
- Проверяйте коды возврата: `if [ $? -eq 0 ]; then`

### Форматирование:
- Отступы: 4 пробела
- Максимальная длина строки: 100 символов
- Пустая строка в конце файла

### Комментарии:
```bash
# Краткое описание функции
function_name() {
    # Подробное описание действия
    command
}
```

## Тестирование

Перед отправкой PR протестируйте:

```bash
# Проверка синтаксиса
bash -n system-updater.sh

# Проверка с shellcheck (если установлен)
shellcheck system-updater.sh

# Запуск на Ubuntu
sudo ./system-updater.sh
```

## Что приветствуется

- Исправление ошибок
- Улучшение документации
- Добавление новых функций
- Оптимизация кода
- Перевод на другие языки
- Улучшение интерфейса

## Что не принимается

- Код без комментариев
- Изменения без тестирования
- Нарушение стиля кода
- Добавление зависимостей без необходимости
